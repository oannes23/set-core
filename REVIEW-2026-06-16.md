# SET.crawl — State-of-Project Deep Dive (2026-06-16)

*A four-agent fan-out review: code reality vs. design docs, FABLE bug-ledger verification,
and a market/genre analysis. Health baseline at time of writing: **178/178 tests pass,
`tsc --noEmit` clean**, ~8.1k LOC engine/core/ui + ~2.4k LOC tests.*

---

## 0. Headline

**The project is in materially better shape than its own docs claim.** The ROUNDS v3
rewrite plus the 2026-06-09/10 fix pass closed essentially every concrete engine bug from
the FABLE review, all six hard invariants are upheld with passing tests, and the run economy
(delve → loot → gold → gear → XP → level-up → tithe → persistence) is **playable
end-to-end** — even though several code comments and a couple of doc lines still describe it
as a placeholder.

The three things that actually need attention are not bugs:
1. **The combat is "too easy for skilled play"** — the central, documented balance tension, gated on a sim that hasn't run.
2. **`ui/app.ts` is a 3,203-line untested monolith** holding all the load-bearing run-economy glue.
3. **Accessibility (colorblind redundant encoding) is unbuilt** — and for a game where *color is a match axis*, that is a pre-public blocker, not a nicety.

## 1. What is actually built (code reality)

### Maturity by module
- **`core/` (Set + generator)** — *solid, fully tested.* AG(f,3) primitives, lock-aware set
  counting, pure spec→board generator with rejection sampling + deterministic fallback +
  floor guarantee. This is the heavily-validated heart; trust it.
- **`data/` (content)** — *solid and rich.* 2 full dungeons (Goblin Warren, Emberdeep) + a
  template variant + tutorial/gauntlet; ~25 creatures, ~20 variants, traps/drifts, 9 classes,
  15 gear bases. Some affix families are **authored-but-staged** (`live:false`, never minted yet).
- **`engine/` (DOM-free reducer)** — *solid, deep, the center of gravity.* `combat.ts` (503
  lines: Rounds v3, rollover exchange, crit, combo, dread, procs, kill-race), `resolve.ts`
  (opposed-stat contests, telegraph, dodge), `triggers.ts` (trap bus, board verbs, wound law,
  selection protection), plus loot/foe/run/session/abilities/consumables — all real and tested.
  `foe.ts` computes XP/gold/level from the statline.
- **`ui/app.ts`** — *solid but a 3.2k-line monolith (~31% of the codebase), and untested.*
  Holds the scene router, town, combat play screen, the Mörk-Borg breakdown cutscene, the
  delve fork, loot banking, tithe, XP award, and persistence writes.

### Playable end-to-end *right now*
Town / character select → dungeon select → **two combat paths** (practice/gauntlet or a full
multi-room delve with boss inverse-CDF + elite sawtooth + carried HP/satchel) → full Rounds-v3
combat with gear folded in → end beat: lone-fight win/defeat ledger **or** the delve fork
(loot reveal → press-on/cash-out with dread pips; **boss kill = dungeon cleared + marquee
gear**; **death = lose satchel + carried gold + ~12% bank tithe**). Gold banks on safe exit,
gear banks to Storage, roster persists across sessions with versioned save migrations.

**Not yet present:** a shop (the gold *sink*), spellbook drops, gear alt-verb affixes, gear
color-scoping, the gear smith, and the entire Rounds-v3 **UI** layer (the engine is built;
the Tactics wheel / round bar / rollover choreography / wound-row / sated cue are not).

## 3. FABLE bug-ledger — verified status

The ROUNDS v3 rewrite dissolved most of FABLE's engine bug sites entirely.

**Closed (verified in code):** E1 (clock-cap windfall), E2 (post-death `lost` re-emit), E3
(`highest_mag` no-op), E4 (match-fired traps cancelled), E5/E6 (actions after combat over),
E8 (generator crash/hang edges), U1 (Enter confirms danger), U2 (modal listener leak), U4
(bad-trio lingering), the makeable-floor invariant risk (I1), and the Goblin King
`war_cry` / dead elite-mirror doc bugs.

**Still live (all low-impact, mostly FABLE-deferred):**
1. **E7 — scroll vs. ethereal foe deals silent 0 damage.** `consumables.ts:200` bypasses
   `castDamageHook`; the ethereal goblin still ships in the gauntlet. A damage scroll does
   literally nothing, with no feedback. Fix: route scrolls through the cast hook, or at least
   emit an `immune` event so the UI can explain the zero.
2. **U5 — `V.actions` records every tick uncoalesced** (`app.ts:1511`). Harmless until the
   `session.ts` replay seam or daily-seeded runs ship, then it's replay bloat. Prerequisite for
   the daily-run feature.
3. **I4 — silent below-floor fallback** (`generate.ts:144/227/258`). No dev assert/canary. The
   invariant sweeps make a real break unlikely, but a prod break would be invisible. Cheap to add.
4. **U6 residual** — a *deliberate player cast* that rewrites a selected card in place isn't
   revalidated by key (line 1516 keys on null/locked only). Narrow; hard-rule #6 covers the common case.
5. **A1 — presentation strings baked into engine events** (English + emoji `label`s). Architecture
   debt that bites i18n / server-authority / multi-client later. Not a bug.

**Hard invariants — all six upheld**, each backed by a passing test: no duplicate cards, ≥FLOOR
sets on settled boards, dropped axes pinned, generator is a pure function of spec, aggregate-control,
selection-protected turnover (rule #6), and the lock-layer makeable-set floor.

## 4. Market & genre analysis

### Where it sits
A hot but **saturated** genre (1,440 roguelike-deckbuilders on Steam; documented buyer fatigue)
with **one genuinely uncontested hook**. The category is proven at the top (Slay the Spire 3.7M+,
Balatro 5M+) and dominated by **solo / tiny teams shipping a single legible novel verb**, then
growing via EA patches, free-demo virality, and — decisively — **awards/featuring** (Balatro
added ~1.5M copies the month after The Game Awards).

### The Set mechanic — commercial whitespace with a comprehension tax
- **No commercial video game is built on Set.** Only faithful tabletop ports exist (SET Pro HD,
  ~$4.99, small). Genuinely uncontested territory.
- **Credibility halo:** Mensa Select 1991, classroom-ubiquitous (it literally teaches AG(4,3)),
  a daily puzzle, a national championship — but **not a household name**, and it carries a higher
  cognitive load than match-3's zero-comprehension entry. The deciding variable is **onboarding**.

### Real-time pressure as combat — works, but specific failure modes
*10000000 / You Must Build a Boat* prove the frantic-timed-puzzle-as-dungeon loop can be loved,
but it **polarizes**. Documented mitigations, most already partly aligned with the design:
- **Clock-starts-on-first-action** (Puzzle Quest 3) — lets players plan; cheap, high-value.
- **Graceful miss-degradation** (Paper Mario action commands) rather than hard failure.
- **Speed × RNG = perceived unfairness** — the exact bad-feel **hard-rule #6 already defends
  against. Keep that invariant.**
- **Accessibility is the highest-risk item for *this specific* mechanic:** color is load-bearing
  (a feature that defines a valid line), so colorblind players literally cannot perceive a set if
  hue is the only cue. A round-length slider + color-redundant encoding are table-stakes (Game
  Accessibility Guidelines "Basic" tier).

### Go-to-market
- **Price $9.99–$14.99** (the high-review healthy-seller cluster). $4.99 undersells; $20+ needs brand/volume.
- **Web/PWA is a funnel, not the revenue center** — no built-in discovery, can't be a paid app;
  Steam holds ~75% of PC distribution. Treat web/PWA as a **free demo → Steam wishlist**.
- **itch.io → Steam is a proven pipeline** (e.g. *Die in the Dungeon*), but you need a clear
  fidelity jump between free and paid.
- **Wishlist mechanics:** ~7k = Popular Upcoming; ~10% convert to first-week sales; **Next Fest
  is a multiplier, not discovery** — enter with momentum, stand the page up early.
- **Sessions:** target ~10–15 min runs; **guarantee a persistent meta-reward every run (win or
  lose)** and bulletproof pause/resume for PWA/mobile.

### Ranked killer-feature opportunities
1. **Lean on the Mörk Borg / Persona art identity as *the* marketable hook** — the most defensible asset in a fatigued genre.
2. **A juicy, streamable resolution moment** — pair the deterministic reveal with maximal payoff (the breakdown cutscene is the right instinct; make it clip-worthy).
3. **Reachable "broken build" euphoria *around* the trap layer** — the counter-layer must be outsmartable into a combo high, not merely endured.
4. **Daily seed + leaderboards** — deterministic resolution makes clean, RNG-complaint-free leaderboards nearly free; a strong social lever for a solo game.
5. **Build variety via *unlocks*, not stat grinds** — turn 9 classes + affixes into distinct deep paths; persistent power grinds get panned.
6. **Accessibility + tunable time pressure** (relaxed/no-timer mode, color redundancy, clock-on-first-action) — widens audience and de-risks the timer for the deliberate-RPG crowd.

### Ranked risks / red flags
1. **Comprehension barrier / feature soup** — Set geometry + traps + dread + Tactics + affixes is a steep wall. Ruthless progressive disclosure is mandatory; "players didn't get it" has killed comparable games at demo stage.
2. **AI-art perception** — *if any displayed art is AI-generated*: ~85% of gamers are negative toward Gen-AI in games, concentrated in exactly this audience; review-bomb risk on the hook that sells the game. Keep player-facing art unmistakably human (as Mörk Borg itself is); AI for code is fine and needs no Steam disclosure.
3. **Saturation + low single-player wishlist ceiling** — top converters are co-op; expect a grind, lean on the hook + niche press/streamers.
4. **Solo-dev scope creep** — a 9-class affix-gear multi-system roguelite is the textbook snowball; lock scope, ship content over systems.
5. **Total loot-loss frustration in the dread loop** — pure loss reads as "waste of time"; the cash-out/flee valve helps, but ensure every run advances *something* persistent.

## 5. Recommended next moves (synthesis)

**Tier 1 — unblock the core experience**
- **Run the balance sim and resolve "too easy for skilled play."** This is the single most
  important open item the docs flag. The candidate levers are designed (Heat dial, the
  `X/(X+K)` base-curve question, the foe-difficulty raise that's already built) but explicitly
  deferred to a coupled sim pass — *do not hand-edit §5.6 before running it.*
- **Build the Rounds v3 UI.** The engine has been ready since 2026-06-11; the player can't
  *feel* the round grammar (Tactics wheel, round bar, rollover choreography, wound-row, sated
  cue) without it. This is the biggest experience gap between code and design.

**Tier 2 — de-risk for anyone but the dev**
- **Colorblind redundant encoding + a round-length / relaxed-mode slider.** Pre-public blocker
  given color is a match axis. Cheap relative to impact and table-stakes for the genre.
- **Carve testable logic out of `ui/app.ts`.** The delve fork, loot banking, tithe, and XP award
  are load-bearing and have *no automated safety net*. Extract to a thin testable module.

**Tier 3 — cheap hygiene**
- Delete the dead `engine/delve.ts:rollDelveLoot` placeholder + its test; fix the stale
  "PLACEHOLDER" comments.
- Fix E7 (scroll vs. ethereal — route through the cast hook or emit `immune`).
- Add a dev-only below-floor canary (I4); coalesce tick actions before the replay seam ships (U5).

**Strategic**
- Treat web/PWA as a **free demo funneling to a Steam wishlist**, price **$9.99–$14.99**, stand
  up the Steam page early, and make the resolution moment **clip-worthy** for streamer-driven
  discovery. Keep all player-facing art human-made.
